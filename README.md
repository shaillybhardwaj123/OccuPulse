## рҹ“Ў What Is OccuPulse?

OccuPulse is an **IoT-powered exam hall occupancy monitor** that uses a single ultrasonic sensor on an Arduino UNO to intelligently detect entries and exits вҖ” no second sensor needed. The live count streams over USB serial to a **Flask web backend**, syncs to **Firebase Realtime Database**, and renders on a stunning **dark-mode web dashboard** with charts, stats, and alerts.

Whether you're managing a 30-seat exam room, a library pod, or a lab вҖ” OccuPulse gives you full situational awareness with minimal hardware.

## рҹ§  How the Single-Sensor Algorithm Works

> OccuPulse uses a **timed dual-trigger** method to detect both entries AND exits with just one sensor:

```
  Person approaches doorway
           в”Ӯ
           в–ј
  Distance < 20 cm?
           в”Ӯ
    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”җ
   YES            NO
    в”Ӯ              в”Ӯ
    в–ј              в–ј
First trigger?   Reset state
    в”Ӯ
 в”Ңв”Җв”Җв”ҙв”Җв”Җв”җ
YES    NO (within 3s window)
 в”Ӯ      в”Ӯ
 в–ј      в–ј
ENTRY  EXIT
count++ count--
```

**In plain English:**

- рҹҡ¶ **1st detection** вҖ” someone crosses the threshold вҶ’ **Entry** (count +1)
- рҹҡ¶ **2nd detection within 3 seconds** вҖ” same person steps back вҶ’ **Exit** (count вҲ’1)
- вҸұпёҸ **After 3 seconds** вҖ” state resets; next detection starts fresh

This makes a single sensor surprisingly robust for low-to-medium traffic doorways like exam hall entrances.

## рҹҡҖ Setup & Installation

### Step 1 вҖ” Flash the Arduino

1. Open `arduino/occupancy.ino` in [Arduino IDE](https://www.arduino.cc/en/software)
2. Select **Board:** Arduino UNO
3. Select your port (e.g. `COM3` on Windows, `/dev/ttyUSB0` on Linux)
4. Click **Upload в–¶**

---

### Step 2 вҖ” Set Up Firebase

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project
3. Enable **Realtime Database** вҶ’ Start in **test mode**
4. Go to **Project Settings вҶ’ Service Accounts вҶ’ Generate new private key**
5. Save the downloaded file as `serviceAccountKey.json` in the project root

> вҡ пёҸ **Security Warning:** `serviceAccountKey.json` contains your private Firebase credentials.
> **Never commit it to Git.** Add it to `.gitignore` immediately:
> ```
> echo "serviceAccountKey.json" >> .gitignore
> ```

6. Open `firebase_config.py` and update:

```python
'databaseURL': 'https://YOUR-PROJECT-ID.firebaseio.com'
```

---

### Step 3 вҖ” Install Python Dependencies

```bash
pip install -r requirements.txt
```

`requirements.txt` includes:
```
flask
flask-cors
pyserial
firebase-admin
```

---

### Step 4 вҖ” Configure Serial Port

Open `app.py` and set the correct port for your OS:

```python
# Windows
SERIAL_PORT = 'COM3'

# Linux / Raspberry Pi
SERIAL_PORT = '/dev/ttyUSB0'

# macOS
SERIAL_PORT = '/dev/cu.usbmodem14101'
```

---

### Step 5 вҖ” Run OccuPulse

```bash
python app.py
```

---

### Step 6 вҖ” Open the Dashboard

```
http://localhost:5000
```

## рҹӣ пёҸ Troubleshooting

| Problem | Solution |
|---|---|
| `SerialException: could not open port` | Check `SERIAL_PORT` in `app.py`, ensure Arduino is plugged in |
| Dashboard shows `в—Ҹ DISCONNECTED` | Serial thread retries every 5s automatically; check cable |
| Count never increments | Ensure sensor is within ~50cm of doorway; check TRIG/ECHO wiring |
| Firebase errors on startup | Verify `serviceAccountKey.json` is present **locally** (never commit it) and `databaseURL` in `firebase_config.py` is correct |
| Count goes negative | Normal edge case; the reset button restores to zero |
| Buzzer won't stop | Hall reached max capacity; reduce count or press Reset |
